WordPress 小聚筆記：繞路從來都不是繞路

這場 WordPress 小聚很特別。講者 Moksa 我不是第一次聽——之前在台中、彰化的場次聽過他分享，主題都很硬：自動化工具、部署、n8n，那種要打開電腦跟著做的內容。但這次主辦特地跟他說：不要講技術，講你的人生職涯軌跡，還有你怎麼看 AI 對這個時代的衝擊。

結果這是我今年聽過最不焦慮的一場 AI 分享。

一條完全不像履歷的履歷

Moksa 開場就說「我自己的經歷也沒什麼特別」，然後講了一條大概沒有任何職涯顧問會建議你走的路：

大學讀職能治療，因為覺得台灣醫療場所的環境不夠友善，發現自己其實更擅長分享跟教學。大二大三跑去參加字幕組（鳳凰天使 TSKS），在還沒有串流平台的年代，很多韓國綜藝的影片都是他經手的。跟幾個陌生人一起經營 Running Man 的粉絲專頁，做到快一百萬粉；台灣在地的偶像見面會，那時候基本上每一場都是他們主辦。

因為做影片剪輯，接觸到 FFmpeg，還參與了系統的優化。做到最大的時候，台灣 Fox 電視台一段時間的韓國相關翻譯，是他的小型工作室在做。同一個時期，因為要架站，他碰到了 WordPress 跟 Discuz 論壇——這是他第一次接觸網站。

畢業當兵，退伍後做國外業務，跟國外廠商客戶對接，一做快五年。疫情期間公司營運出問題，一路縮編到剩四個人，但每個月營業額還有四五千萬，因為貨全部卡在海上。硬撐到 2021 年結束。

然後跑去考試，不小心考上成大心理研究所。上了之後還在想「我當初為什麼想做這個」，發現好像不是自己要的。

疫情帶來電商跟代購的商機，他進了一間保健食品公司當軟體工程師，幫他們把舊網站換成 WordPress。然後——用他自己的話——「因為有 AI 的協助，公司根本不需要這麼多人，一不小心就把自己也給拆掉了」。

現在是接案的小型工作室，做 AI 顧問。

三句話

他最後收在三句話上，我覺得這才是整場的重點。

第一句：繞路一直都不算是繞路。

看完上面那條線，你很難說哪一段是「浪費」。字幕組的 FFmpeg 變成後來的技術底子，職能治療跟心理所變成他理解人的方式，外貿業務讓他知道怎麼跟客戶談。他的說法是：繞路反而是在尋找自己的方向，你要先繞過，才會知道自己真正想要的目標是什麼。

第二句：先做再想，不要停在想的部分。

「我對這個東西有興趣我就先去做，我不要去想它要一百年的呈現。我做了之後我就可以了解它是怎麼樣的一個運作方式，做了之後你才知道這個東西到底是不是你想要的。」

第三句：在工作裡持續找樂趣。

這句聽起來像雞湯，但他舉的例子很好笑——有時候客戶想要某個版面，他覺得無聊就故意不給，先套一個別的，等客戶反饋再改回去。重點不是惡作劇，是「不要永遠在同一個工作模式裡，做久了做快了就會變得非常無聊」。

關於 AI：按用途選，不要按聲量選

問答時他被問到工具選擇，回答得很乾脆。他自己固定用三家：

  • ChatGPT / Codex——寫 code，或純聊天、要情緒價值的時候
  • Anthropic 的 Claude——寫 code
  • Google 的 Gemini——要在地化（台灣的語言）、圖像辨識

重點是那句：「這三個工具你要看的是你的用途是什麼，而不是覺得這個 AI 大家都在講，可是實際上你根本用不上。也不是每個人都需要去寫 code，所以你也根本不需要去用 Claude。」

他也提到一個我很認同的判斷：AI 下一個大跳躍會在視覺。目前圖像跟影片生成最關鍵的問題是 AI 在視覺上有沒有幻覺、判斷準不準。他預期今年年底前會有更完整的視覺模型出來——AI 像是有了眼睛，可以看懂圖像、理解影片。各家現在都在往這個方向努力。

n8n 跟 AI 是兩件事

有人問「n8n 現在還需要學嗎？」他的回答很清楚：

n8n 是自動化工具。你的日常生活裡繁複的東西可以留給它做。那你今天要想一個新的想法、新的設計，那就是透過 AI。這兩個工具本來就不應該合併去考慮。

他舉的例子：小聚活動透過 Meetup API 接收新活動、自動發佈——這是 n8n 的活。但「我 7/15 有個工作，我想調整到 7/16」——這是 AI 的活。n8n 做最多的是報表這種重複性極高的東西，另外聊天機器人也很適合。

他真的把東西送出來了

分享中他 demo 了一個自己寫的 Elementor Skill（給 Claude 用的）：用 Python 腳本裝 Elementor 的元件檔，把快九萬多組元件跟設定全部餵進去，跑過驗證。他的官網就是實驗場——所有頁面他說「我都沒有進到後台過」，直接讓 AI 透過這個 Skill 去編輯。之後還打算再加匯入匯出的功能。

他說有興趣的可以直接去 GitHub 抓。

這件事跟他另一個立場是連著的：他把非常多 AI 教學都放在自己的網站上，免費、沒有廣告，網站上有六百多篇文章。理由是——

秉持著 WordPress 的開源精神。我不希望在這個成長這麼快的時代，大家還需要花費這麼多時間跟金錢，去投資所有可能未來不應該投資的東西。

最有價值的一段：怎麼不被焦慮行銷綁架

這是我覺得整場最該被記下來的。他說在這個時代大家常常焦慮：「AI 工具這麼多，我們的工作怎麼辦？我們到底需不需要學 AI？」不管你打開 FB 還是 Threads，演算法時不時就推 AI 課程、AI 場次給你，然後你看那個費用真的很誇張。

他的吐槽很精準：

我們要怎樣去避免這些 AI 講師們在打廣告的時候給你的焦慮感。「UIUX 不行啊，再不學 AI 你可能就會被淘汰」、「前端工程不行啊」——不行在哪裡他也講不出來。

「不行在哪裡他也講不出來」——這句話幾乎可以拿來當所有焦慮行銷的照妖鏡。講不出具體的失效點，那就只是在賣課。

他自己也自嘲：當初在台中小聚講了快八個小時的 n8n，「結果該睡的睡、該躺的躺」。所以這次改講人生議題。

我的收穫

  1. 技術素養會過期，但把自己重新定義的能力不會。 Moksa 的線裡每一段技術都被時代換掉過——字幕組沒了、Discuz 沒了、外貿公司收了、連他自己都被 AI 優化掉。留下來的是那個「先去做做看」的動作本身。

  2. 「不行在哪裡」是個好問題。 下次看到任何販賣淘汰焦慮的東西，就問這一句。答不出來就可以關掉了。

  3. 工具按用途選。 我自己也有那種「大家都在用所以我也該學」的反射，這是該戒掉的。

  4. 繞路那句我想放久一點。 我看自己的路也是東拼西湊的，聽完之後比較不會覺得那是需要辯解的事。

順帶一提，他分享字幕組年代的三個工具讓我蠻懷念的：Aegisub（字幕，可以寫自動化腳本，但上手難度太高所以沒人用）、FFmpeg（轉檔）、MeGUI（字幕跟影片合併，一開下去 CPU 直接 99%）。他還提到一個冷知識：那個年代的 720P 畫質會比現在的 1080P 好很多，因為現在的編碼為了效率犧牲掉太多東西。
